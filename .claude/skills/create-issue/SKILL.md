# create-issue

사용자의 설명을 바탕으로 GitHub 이슈를 자동 생성한다.

## 레포지토리 정보

- owner: `jxmen`
- repo: `commerce-example`

## 동작 순서

1. 사용자의 설명을 분석하여 제목, 본문, 라벨을 결정한다.
2. `mcp__github__create_issue` 도구로 이슈를 생성한다.
3. 이슈 번호를 사용해 로컬 git 브랜치를 생성한다.
4. 생성된 이슈의 제목, 본문, 라벨, URL과 브랜치명을 사용자에게 보여준다.

## 브랜치 생성 규칙

이슈 생성 후 아래 순서로 로컬 브랜치를 만든다:

1. 브랜치 생성 및 체크아웃: `git checkout -b feature/<이슈번호>`
2. 브랜치 설명 설정 (이슈 제목만 포함):
   ```bash
   git config branch.feature/<이슈번호>.description "<이슈 제목>"
   ```

## 제목 작성 규칙

- 간결하고 명확하게, 50자 이내
- 한국어 설명이면 한국어로 작성
- 동사형으로 끝내기 (예: "상품 목록 조회 API 구현", "결제 버그 수정")

## 본문 작성 규칙

아래 구조로 작성한다:

```
## 개요
[기능/문제에 대한 간략한 설명]

## 작업 내용
- [ ] [세부 작업 항목 1]
- [ ] [세부 작업 항목 2]

## 참고 사항
[관련 컨텍스트, 주의사항 등 — 없으면 이 섹션 생략]
```

## 라벨 선택 규칙

사용자가 라벨을 명시하지 않으면 `enhancement`를 기본으로 사용한다.

| 상황 | 라벨 |
|------|------|
| 새 기능, 기능 개선 (기본) | `enhancement` |
| 버그, 오류, 예외 처리 | `bug` |
| 문서, 주석, README | `documentation` |
| 질문, 논의, 결정 필요 | `question` |

## 결과 출력 형식

이슈 및 브랜치 생성 후 다음 형식으로 출력한다:

```
**제목:** [제목]
**라벨:** [라벨명]
**URL:** [이슈 URL]
**브랜치:** feature/<이슈번호>

**본문:**
[생성된 본문 내용]
```
