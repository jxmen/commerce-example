## 작업 순서

### 1단계: 변경 파일 목록 확인

`git status --short`로 변경된 파일 전체를 사용자에게 보여준다.

```
변경된 파일 목록:
 M src/main/java/dev/jxmen/commerce/domain/Seller.java
?? src/main/java/dev/jxmen/commerce/interfaces/controller/SellerController.java
```

### 2단계: 파일 선택

사용자에게 어떤 파일을 커밋에 포함할지 묻는다.

예시:
> 위 파일 중 커밋에 포함할 파일을 알려주세요. (전체 포함이면 "전체" 또는 "all")

사용자가 선택한 파일만 `git add`로 스테이징한다.

### 3단계: 변경 내용 분석

스테이징된 파일들의 diff를 확인한다:

```bash
git diff --staged
```

변경 내용을 바탕으로 **무엇을**, **왜** 변경했는지 파악한다.

### 4단계: 커밋 메시지 생성

아래 규칙으로 한국어 커밋 메시지를 작성한다:

**제목 (50자 이내)**
- 변경의 핵심을 간결하게 한 줄로 표현
- 동사로 시작 (추가, 수정, 삭제, 리팩토링 등)
- 예: `셀러 CRUD API 추가`

**본문 (선택, 72자 이내로 줄 바꿈)**
- 제목과 빈 줄 하나로 구분
- 변경한 이유나 배경, 주요 변경 사항을 설명
- 간단한 변경이면 본문 생략 가능

### 5단계: 커밋 실행

```bash
git commit -m "제목" -m "본문"
```

### 6단계: 결과 출력

커밋 후 아래 형식으로 결과를 출력한다:

```
커밋 완료!

제목: 셀러 CRUD API 추가
본문:
셀러 등록, 조회, 삭제 엔드포인트를 추가했습니다.
클린 아키텍처 레이어 구조(interfaces/application/infrastructure)에 맞게 구현했습니다.

수정이 필요하면 `git commit --amend`로 변경할 수 있습니다.
```

사용자가 메시지를 수정하고 싶다고 하면 `git commit --amend`를 안내한다.
